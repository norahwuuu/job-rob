# 项目结构速览（中文导读）

这份文档用于快速理解这个仓库“有哪些模块、谁调用谁、出问题先看哪里”。

## 1) 这个项目在做什么

目标：根据岗位描述（JD）自动优化 Word 简历，并导出可投递的 PDF。

整体由两部分组成：

- `Word Editor/`：Python 后端，负责 AI 分析、Word 修改、PDF 导出、日志记录。
- `web-to-pdf-plugin/web-to-md-pdf-plugin/`：浏览器插件，负责提取网页职位内容并调用后端 API。

## 2) 一条请求的完整链路

1. 用户在浏览器点击插件按钮。
2. 插件提取页面中的岗位信息，调用后端 `POST /api/modify-resume`。
3. 后端创建任务并在线程池中异步执行：
   - 解析简历文本
   - 调用 AI 生成修改指令
   - 把指令应用到 `.docx`
   - 可选导出 `.pdf`
4. 后端返回任务状态；插件轮询状态并显示结果。
5. 用户下载命名后的 Word/PDF 文件，并可在历史日志中查看记录。

## 3) 后端关键文件（先看这些）

- `Word Editor/resume_modifier/api_server.py`
  - FastAPI 入口，任务管理、状态查询、下载接口、日志接口。
- `Word Editor/resume_modifier/main.py`
  - 核心编排函数 `process_resume()`，贯穿解析 -> AI -> 修改 -> 导出。
- `Word Editor/resume_modifier/ai_analyzer.py`
  - Prompt、模型调用、JSON 容错解析、输出指令后处理。
- `Word Editor/resume_modifier/content_modifier.py`
  - 真正修改 Word 内容的地方，重点在 paragraph/run 粒度和格式保留。
- `Word Editor/resume_modifier/resume_parser.py`
  - 从 Word 文档提取可供 AI 理解的文本。
- `Word Editor/resume_modifier/job_log.py`
  - 申请日志持久化与统计。

## 4) 常见概念（看日志时会遇到）

- `target`：要匹配的原文（必须来自简历真实文本）
- `replacement`：替换后的文本
- `match_type`：
  - `fuzzy`：默认，容错匹配
  - `add_after`：在某段后面插入
  - `replace_paragraph`：整段重写
- `match_score`：AI 估算的岗位匹配度（0-100）

## 5) 出问题先排查哪里

1. **后端是否启动**：`GET /api/health`
2. **AI 是否返回有效 JSON**：`ai_analyzer.py` 的 `_parse_json_safely()`
3. **为什么“未找到目标文本”**：`content_modifier.py` 的匹配逻辑（`_find_match`）
4. **为什么文件保存失败**：是否被 Word 占用（`main.py` 有锁文件检测）
5. **为什么下载不到文件**：检查 `/api/task-status/{task_id}` 与 `/api/download/...`

## 6) 建议阅读顺序（30 分钟版本）

1. 读 `Word Editor/README.md`（看功能和运行方式）
2. 读 `api_server.py`（看接口与任务生命周期）
3. 读 `main.py`（看业务主流程）
4. 读 `ai_analyzer.py` + `content_modifier.py`（看“智能 + 落地”核心）
